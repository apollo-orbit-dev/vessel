# Linux desktop integration for `.vessel`

A `.vessel` is a ZIP archive, so Linux desktops content-sniff it as
`application/zip` — which means the file manager won't let you associate the
`.vessel` extension on its own (you can only target the generic zip type), and
`.vessel` files show the zip icon.

This registers a proper **`application/vessel`** MIME type — a subclass of
`application/zip` matched by the `*.vessel` glob, so the desktop prefers it for
`.vessel` files — plus the Vessel document **icon**. It's a per-user install
(`~/.local/share/...`); no root required.

> Note: the Vessel host is installed as a PWA (from getvessel.dev), and Chrome's
> PWA install doesn't teach the Linux MIME database about `.vessel`. This bridges
> that gap. A future native (Tauri) build would register all of this at install
> time automatically.

## Install

```bash
./install.sh
```

Then verify and set the Vessel host as the default opener:

```bash
xdg-mime query filetype some.vessel        # -> application/vessel
app=$(ls ~/.local/share/applications/ | grep -i vessel | head -1)
xdg-mime default "$app" application/vessel
```

Restart the file manager (`nautilus -q`, it relaunches on next use) or log
out/in so the new type + icon are picked up.

## Uninstall

```bash
./install.sh --uninstall
```

## Files

- `vessel.xml` — the shared-mime-info definition.
- `icons/application-vessel-{16,32,48,256}.png` — the `.vessel` file icon
  (the same document icon the host declares, generated by
  `host/scripts/gen-icons.mjs`).
- `install.sh` — registers/removes the MIME type + icons for the current user.
